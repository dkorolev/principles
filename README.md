# Principles

Codified, repo-agnostic engineering doctrine. Each document is a living set of imperatives — written to be checked, enforced, and argued about, by humans and coding agents alike. Product features appear only as examples; nothing here is specific to any one repository.

## The documents

Two, so far:

- **[`ENG-PRINCIPLES.md`](ENG-PRINCIPLES.md)** — how we build: strong typing with one source-of-truth language, CLI ergonomics and machine output, testing and executable Markdown harnesses, docs, and more.

- **[`WEB-UI-PRINCIPLES.md`](WEB-UI-PRINCIPLES.md)** — UI, UX, and data doctrine for web applications, document-style local apps in particular: local-first, offline-first, CRDT-first; stability; direct interaction; progressive disclosure; visible-data integrity; accessibility.

## How to use them

In each repository, reference the relevant documents from `AGENTS.md` / `CONTRIBUTING.md` (or the agent-config file the repo uses), so both humans and coding agents load them as house rules. Link, don't copy — this repo is the one source of truth, and copies drift.
